# Claude Code Spec Development Template

このレポジトリは、Claude Codeを使用した仕様駆動開発を行う際のテンプレート資料を格納したものです。

## 目的

新規プロジェクトを開始する際に、このレポジトリをコピーして使用することで、効率的にAIによるコーディングを進めることができます。各種ドキュメントを参照させながら、一貫性のある開発を実現します。

## 使用方法

1. このレポジトリを新規プロジェクトのディレクトリにコピー
2. プロジェクトの要件に合わせて各ドキュメントを編集
3. Claude Codeにドキュメントを参照させながら開発を進行
4. 必要に応じてドキュメントを更新・追加

## ディレクトリ構造

```
.
├── README.md          # このファイル
└── docs/              # プロジェクトドキュメント
    ├── 00-rules/      # 開発ルール・規約
    ├── 01-requirements/  # 要件定義
    ├── 02-architecture/  # アーキテクチャ設計
    ├── 03-features/      # 機能仕様
    ├── 04-api/           # API仕様
    └── 05-tests/         # テスト仕様
```

詳細は [docs/README.md](docs/README.md) を参照してください。

## 推奨される開発フロー

1. `00-rules` - プロジェクトの開発ルールとコーディング規約を定義
2. `01-requirements` - ビジネス要件とシステム要件を明確化
3. `02-architecture` - システムアーキテクチャと技術スタックを決定
4. `03-features` - 各機能の詳細仕様を記述
5. `04-api` - API設計とインターフェース定義
6. `05-tests` - テスト戦略とテストケースを策定

## ドキュメント管理について

このプロジェクトは **Gitによるバージョン管理** を前提としています。

### 重要事項

- **ドキュメント内に改訂履歴や変更履歴を記載する必要はありません**
- すべての変更履歴は **Gitのコミット履歴** で管理されます
- ドキュメントは常に **最新の状態** を保ち、過去のバージョン情報はGitで参照してください
- ドキュメントの変更は適切なコミットメッセージとともにコミットしてください

### 📊 図表管理

プロジェクト内の図表は用途に応じて異なる方法で管理されています：

#### 技術的な図表（フローチャート、ER図、シーケンス図など）
- **作成方法**: Mermaid記法でMarkdown内に直接記述
- **ファイル形式**: Markdown内にテキストベースで埋め込み
- **詳細ルール**: [docs/00-rules/mermaid-diagram-rules.md](docs/00-rules/mermaid-diagram-rules.md)

#### 画面設計（ワイヤーフレーム、UIモックアップ）
- **作成ツール**: draw.io（[app.diagrams.net](https://app.diagrams.net/)）
- **ファイル形式**: `.drawio`（編集用）と`.png`（表示用）のペア
- **配置場所**: `docs/03-features/screens/`ディレクトリ
- **詳細ルール**: [docs/00-rules/screen-design-rules.md](docs/00-rules/screen-design-rules.md)

### Git管理のメリット

- 変更履歴が自動的に記録される
- 誰がいつ何を変更したか明確
- 過去の任意の時点の状態に戻すことが可能
- ブランチを使った並行作業が可能
