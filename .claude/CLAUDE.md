# Claude Code プロジェクトガイド

## プロジェクト概要

このレポジトリは、Claude Codeを使用した仕様駆動開発を行う際のテンプレート資料を提供します。
新規プロジェクト開始時にこのレポジトリをコピーし、プロジェクト固有の内容に修正して使用します。

## ドキュメント構造

このプロジェクトでは、`docs/` ディレクトリ配下に体系的なドキュメントを配置しています：

- `docs/00-rules/` - 開発ルール・コーディング規約
- `docs/01-requirements/` - 要件定義
- `docs/02-architecture/` - アーキテクチャ設計
- `docs/03-features/` - 機能仕様
- `docs/04-api/` - API仕様
- `docs/05-tests/` - テスト仕様

詳細は `docs/README.md` を参照してください。

## 開発時の指針

### コード作成・修正時

1. **必ず関連ドキュメントを確認**
   - `docs/00-rules/` のコーディング規約に従う
   - `docs/01-requirements/` で要件を確認
   - `docs/03-features/` で機能仕様を確認

2. **アーキテクチャ原則の遵守**
   - `docs/02-architecture/` の設計方針に従う
   - 既存のパターンとの一貫性を保つ

3. **API実装時**
   - `docs/04-api/` の仕様に準拠
   - 認証・認可要件を確認

4. **テストの実装**
   - `docs/05-tests/` のテスト戦略に従う
   - 必要なテストケースをカバー

### ドキュメント参照の優先順位

コードを書く前に、以下の順序でドキュメントを確認してください：

1. `00-rules/` - プロジェクト全体のルールを理解
2. `01-requirements/` - 実装する機能の要件を確認
3. `02-architecture/` - 技術的な制約と設計方針を確認
4. `03-features/` または `04-api/` - 詳細な仕様を確認
5. `05-tests/` - テスト要件を確認

## 作業フロー

### 新機能実装時

1. `docs/01-requirements/` で要件を確認
2. `docs/03-features/` または `docs/04-api/` で詳細仕様を確認
3. `docs/00-rules/` のコーディング規約を遵守して実装
4. `docs/05-tests/` に基づいてテストを作成
5. 実装とドキュメントの整合性を確認

### バグ修正時

1. 関連する機能の `docs/03-features/` を確認
2. `docs/00-rules/` のコーディング規約を遵守して修正
3. `docs/05-tests/` に基づいて回帰テストを実施

### リファクタリング時

1. `docs/02-architecture/` の設計方針を確認
2. `docs/00-rules/` のコーディング規約に従う
3. 既存の機能仕様 `docs/03-features/` との整合性を保つ
4. テストが通ることを確認

## 注意事項

- ドキュメントが存在しない場合、まずドキュメントの作成を検討してください
- 実装とドキュメントに乖離がある場合、必ずドキュメントを更新してください
- 不明点がある場合、推測せずにユーザーに確認してください
- このレポジトリはテンプレートのため、空のディレクトリが存在する可能性があります
- **Git管理について**: このプロジェクトはGitによるバージョン管理を前提としています
  - ドキュメント内にバージョン履歴セクションや変更履歴を記載する必要はありません
  - すべての変更履歴はGitのコミット履歴で管理されます
  - ドキュメントは常に最新の状態を保ち、過去のバージョン情報はGitで参照してください
