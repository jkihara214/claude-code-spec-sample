## 概要

このディレクトリには、プロジェクト全体で統一すべき開発ルールとコーディング規約を格納します。
すべてのプロジェクトメンバーは、これらのルールを遵守して開発を進める必要があります。

## ルールの適用

### Claude Codeでの利用

Claude Codeを使用する際は、これらのルールを参照するよう指示してください：

```
docs/00-rules/diagram-creation-rules.md に従って図表を作成してください
```

### チーム内での共有

1. 新規メンバー参加時に必ず確認
2. 定期的なルールレビュー会の実施
3. ルール違反時の指摘と改善

## ルールの更新

### 更新プロセス

1. **提案**: GitHubのIssueで提案
2. **議論**: チームで議論
3. **承認**: 技術リーダーの承認
4. **更新**: ドキュメント更新とプルリクエスト
5. **周知**: 全メンバーへの通知

### バージョン管理

- すべての変更はGitで管理
- 重要な変更は変更ログを記載
- 過去のバージョンはGit履歴で参照
