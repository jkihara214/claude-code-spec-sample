# 00-rules - 開発ルール・規約

プロジェクト全体で統一すべき開発ルールとコーディング規約を定義します。
すべての工程で参照される横断的なドキュメントです。

## 目次

- [ディレクトリ構成](#ディレクトリ構成)
- [各ファイルの説明](#各ファイルの説明)

## ディレクトリ構成

```
00-rules/
├── README.md                # このファイル
├── coding-standards.md      # コーディング規約
├── naming-conventions.md    # 命名規則
├── git-workflow.md          # Git運用ルール
└── review-guidelines.md     # レビュー基準
```

## 各ファイルの説明

### coding-standards.md - コーディング規約

言語別のスタイルガイドを定義します。

記載内容:
- 使用言語ごとのコーディングスタイル
- インデント、空白、改行のルール
- コメントの書き方
- ファイル構成のルール

### naming-conventions.md - 命名規則

変数、関数、ファイル等の命名ルールを定義します。

記載内容:
- 変数・定数の命名規則
- 関数・メソッドの命名規則
- クラス・コンポーネントの命名規則
- ファイル・ディレクトリの命名規則
- データベースのテーブル・カラム命名規則

### git-workflow.md - Git運用ルール

ブランチ戦略、コミットメッセージ規約を定義します。

記載内容:
- ブランチ戦略（Git Flow、GitHub Flow等）
- ブランチの命名規則
- コミットメッセージのフォーマット
- プルリクエストのルール
- マージ・リリースの手順

### review-guidelines.md - レビュー基準

コードレビューのチェックリストを定義します。

記載内容:
- レビュー観点（機能、パフォーマンス、セキュリティ等）
- レビューチェックリスト
- レビューコメントの書き方
- 承認基準
