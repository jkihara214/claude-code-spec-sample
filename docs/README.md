# ドキュメント構成

このディレクトリには、Claude Codeを使用した開発プロジェクトに必要な各種ドキュメントを格納します。

## ディレクトリ構成

### 00-rules/
**開発ルール・規約**

プロジェクト全体で統一すべき開発ルールとコーディング規約を定義します。

格納する内容:
- コーディング規約
- 命名規則
- ディレクトリ構成ルール
- Git運用ルール
- レビュー基準
- セキュリティガイドライン

### 01-requirements/
**要件定義**

プロジェクトのビジネス要件とシステム要件を明確に記述します。

格納する内容:
- ビジネス要件
- 機能要件
- 非機能要件
- ユースケース
- ユーザーストーリー
- 制約事項

### 02-architecture/
**アーキテクチャ設計**

システム全体のアーキテクチャと技術的な設計方針を定義します。

格納する内容:
- システムアーキテクチャ図
- 技術スタック選定理由
- データベース設計
- インフラ構成
- セキュリティアーキテクチャ
- パフォーマンス設計

### 03-features/
**機能仕様**

個別の機能に関する詳細な仕様を記述します。

格納する内容:
- 機能詳細仕様
- 画面仕様
- ビジネスロジック
- データフロー
- エラーハンドリング
- バリデーションルール

### 04-api/
**API仕様**

APIのインターフェース定義とドキュメントを管理します。

格納する内容:
- RESTful API仕様
- GraphQL スキーマ
- リクエスト/レスポンス例
- 認証・認可仕様
- エラーコード定義
- API バージョニング方針

### 05-tests/
**テスト仕様**

テスト戦略とテストケースを定義します。

格納する内容:
- テスト戦略
- テスト計画
- 単体テスト仕様
- 統合テスト仕様
- E2Eテスト仕様
- パフォーマンステスト仕様
- テストデータ定義

## ドキュメント作成のガイドライン

1. **Markdown形式で記述**: 全てのドキュメントはMarkdown形式で作成してください
2. **参照の明記**: 他のドキュメントを参照する場合は、相対パスで明記してください
3. **図の活用**: 複雑な内容は図表を用いて視覚化してください
4. **定期的な更新**: 実装と仕様の乖離が発生しないよう、定期的に更新してください

## Claude Codeへの指示例

ドキュメントを参照させる際の指示例:

```
docs/01-requirements/ の要件定義を確認して、
docs/03-features/ の機能仕様に基づいて実装してください。
実装の際は docs/00-rules/ のコーディング規約に従ってください。
```

## 注意事項

- 機密情報を含む場合は、適切なアクセス制御を設定してください
- プロジェクト固有の情報は、このテンプレートをコピー後に編集してください
- ドキュメントの粒度は、プロジェクトの規模に応じて調整してください
