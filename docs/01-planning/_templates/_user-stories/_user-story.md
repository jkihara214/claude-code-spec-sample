# ユーザーストーリー テンプレート

> **使い方**:このテンプレートは `user-stories/US{番号}-{名前}.md` の記述ルールを定義します。
> - 各セクションの「記述ルール」と「記述例」を参考にして内容を記述してください。
> - **出力形式**: 実際に作成するファイルには「記述ルール」と「記述例」は含めず、内容のみを記述してください。
> - **セクションの統一**: 該当しないセクションも削除せず「該当なし」と記載してください。

## 基本情報

### 記述ルール

- ユーザーストーリーの基本的な情報をテーブル形式で記述する
- アクターは [functional-requirements.md](../functional-requirements.md) のアクター定義と整合させる

#### 記述例

| 項目 | 内容 |
|-----|------|
| ストーリーID | US010 |
| 機能名 | 受注登録 |
| アクター | 営業担当 |
| 概要 | 顧客からの注文を受け付け、受注情報を登録する |

## ストーリー一覧

### 記述ルール

- 1機能につき複数のユーザーストーリーを記述できる
- 各ストーリーは連番で管理する（US010-01, US010-02, ...）
- 「As a / I want to / So that」形式で記述する
- 受け入れ条件はチェックリスト形式で記述する

### ストーリー

#### 記述例

**US010-01: 基本的な受注登録**

| 項目 | 内容 |
|-----|------|
| As a | 営業担当 |
| I want to | 顧客からの注文を登録したい |
| So that | 受注情報を管理し、出荷手配ができる |

受け入れ条件:

- [ ] 顧客を選択できる
- [ ] 商品を選択できる
- [ ] 数量を入力できる
- [ ] 納品希望日を入力できる
- [ ] 登録ボタンで受注が保存される
- [ ] 受注番号が発行される

---

**US010-02: 複数商品の一括登録**

| 項目 | 内容 |
|-----|------|
| As a | 営業担当 |
| I want to | 複数の商品を一度に登録したい |
| So that | 入力の手間を削減できる |

受け入れ条件:

- [ ] 商品行を追加できる
- [ ] 商品行を削除できる
- [ ] 合計金額が自動計算される

---

**US010-03: 在庫確認しながらの登録**

| 項目 | 内容 |
|-----|------|
| As a | 営業担当 |
| I want to | 在庫状況を確認しながら注文を登録したい |
| So that | 在庫切れによる納期遅延を防げる |

受け入れ条件:

- [ ] 商品選択時に在庫数が表示される
- [ ] 在庫不足の場合は警告が表示される
- [ ] 代替納期が提示される
