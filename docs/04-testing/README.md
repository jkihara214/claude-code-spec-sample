# 04-testing - テスト設計

品質確保のためのテスト計画・戦略・テストケースを定義します。
V字モデルに基づき、各設計工程に対応するテストを設計します。

## ディレクトリ構成

```
04-testing/
├── README.md            # このファイル
├── test-plan.md         # テスト計画書
├── test-strategy.md     # テスト戦略
├── unit-tests/          # 単体テスト仕様
├── integration-tests/   # 結合テスト仕様
├── e2e-tests/           # E2Eテスト仕様
└── test-data/           # テストデータ定義
```

## V字モデルとの対応

| 設計工程 | 対応するテスト | ディレクトリ |
|---------|--------------|-------------|
| 詳細設計 | 単体テスト | unit-tests/ |
| 詳細設計 | 結合テスト | integration-tests/ |
| 基本設計 | E2E/総合テスト | e2e-tests/ |
| 要件定義 | 受入テスト | e2e-tests/ |

## 各ファイル・ディレクトリの説明

### test-plan.md - テスト計画書

テスト全体の方針・スケジュール・体制を定義します。

記載内容:
- テスト目的・スコープ
- テスト対象・対象外
- テストスケジュール
- テスト環境
- テスト体制・役割分担
- 合格基準
- リスクと対策

### test-strategy.md - テスト戦略

テストレベル・手法・ツール選定を定義します。

記載内容:
- テストレベル（単体、結合、E2E、受入）
- テスト手法（ホワイトボックス、ブラックボックス）
- テスト自動化方針
- 使用ツール
- カバレッジ目標
- 回帰テスト方針

### unit-tests/ - 単体テスト仕様

モジュール単位のテストケースを定義します。

配置するファイル:
- モジュール単位でファイルを作成
- 例: `order-service.test.md`, `validation.test.md`

記載内容:
- テスト対象（関数・メソッド）
- テスト観点
- 入力値と期待結果
- 境界値・異常系テスト

### integration-tests/ - 結合テスト仕様

モジュール間連携のテストケースを定義します。

配置するファイル:
- 処理フロー単位でファイルを作成
- 例: `order-flow.test.md`, `payment-integration.test.md`

記載内容:
- テスト対象（連携するモジュール）
- テストシナリオ
- 入力値と期待結果
- データ整合性確認

### e2e-tests/ - E2Eテスト仕様

エンドツーエンドのシナリオテストを定義します。

配置するファイル:
- ユーザーシナリオ単位でファイルを作成
- 例: `login-scenario.test.md`, `purchase-scenario.test.md`

記載内容:
- 前提条件
- テストシナリオ（操作手順）
- 期待結果
- 確認項目

### test-data/ - テストデータ定義

テスト用のサンプルデータを定義します。

配置するファイル:
- データ種別ごとにファイルを作成
- 例: `users.md`, `products.md`, `orders.md`

記載内容:
- テストデータ一覧
- データ作成方法
- データクリーンアップ方法
