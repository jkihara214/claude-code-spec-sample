## ディレクトリ構成

### 00-rules/
**開発ルール・規約**

プロジェクト全体で統一すべき開発ルールとコーディング規約を定義します。

格納する内容:
- コーディング規約
- 命名規則
- ディレクトリ構成ルール
- Git運用ルール
- レビュー基準
- セキュリティガイドライン

### 01-requirements/
**要件定義**

プロジェクトのビジネス要件とシステム要件を明確に記述します。

格納する内容:
- ビジネス要件
- 機能要件
- 非機能要件
- ユースケース
- ユーザーストーリー
- 制約事項

### 02-architecture/
**アーキテクチャ設計（基本設計）**

システム全体のアーキテクチャと技術的な設計方針を定義します。

格納する内容:
- システムアーキテクチャ図
- 技術スタック選定理由
- データベース設計
- インフラ構成
- セキュリティアーキテクチャ
- パフォーマンス設計

### 03-ui-design/
**UI設計（基本設計）**

個別の画面に関するUI設計を記述します。

格納する内容:
- 画面レイアウト・ワイヤーフレーム
- UIコンポーネント仕様
- バリデーションルール定義
- 画面状態定義
- レスポンシブ対応
- アクセシビリティ要件

### 04-api/
**API仕様（基本設計）**

APIのインターフェース定義とドキュメントを管理します。

格納する内容:
- RESTful API仕様
- GraphQL スキーマ
- リクエスト/レスポンス例
- 認証・認可仕様
- エラーコード定義
- API バージョニング方針

### 05-interactions/
**インタラクション仕様（詳細設計）**

UIとAPIの相互作用、処理フローの詳細を定義します。

格納する内容:
- UI×API紐づけ（マッピング）
- 処理フロー・シーケンス図
- バリデーション実行タイミング
- エラーハンドリングフロー
- 状態管理
- パフォーマンス最適化

### 06-tests/
**テスト仕様**

テスト戦略とテストケースを定義します。

格納する内容:
- テスト戦略
- テスト計画
- 単体テスト仕様
- 統合テスト仕様
- E2Eテスト仕様
- パフォーマンステスト仕様
- テストデータ定義

## 設計フェーズとの対応

| 設計フェーズ | ディレクトリ |
|------------|------------|
| 要件定義 | `01-requirements/` |
| 基本設計 | `02-architecture/`, `03-ui-design/`, `04-api/` |
| 詳細設計 | `05-interactions/` |
| テスト設計 | `06-tests/` |

**作成順序の目安:**
1. `01-requirements/` - 要件を明確化
2. `02-architecture/` - 技術方針を決定
3. `03-ui-design/` と `04-api/` - 並行して作成可能
4. `05-interactions/` - UI設計とAPI設計が概ね固まってから作成
5. `06-tests/` - 各フェーズに応じて作成

## ドキュメント作成のガイドライン

1. **Markdown形式で記述**: 全てのドキュメントはMarkdown形式で作成してください
2. **参照の明記**: 他のドキュメントを参照する場合は、相対パスで明記してください
3. **図の活用**: 複雑な内容は図表を用いて視覚化してください（技術図表は[Mermaid図表記述ルール](00-rules/mermaid-diagram-rules.md)、画面設計は[画面設計ルール](00-rules/screen-design-rules.md)を参照）
4. **定期的な更新**: 実装と仕様の乖離が発生しないよう、定期的に更新してください

### 📊 図表管理について

図表は用途に応じて異なる方法で管理します：

#### 技術的な図表（フローチャート、ER図、シーケンス図など）
- **作成方法**: Mermaid記法でMarkdown内に直接記述
- **管理**: テキストベースでGit管理
- **詳細ルール**: [**Mermaid図表記述ルール**](00-rules/mermaid-diagram-rules.md)を参照

#### 画面設計（ワイヤーフレーム、UIモックアップ）
- **編集用**: `.drawio`ファイル（draw.io形式）
- **表示用**: `.png`ファイル（200%ズームでエクスポート）
- **配置場所**: `docs/03-ui-design/screens/`ディレクトリ
- **詳細ルール**: [**画面設計ルール**](00-rules/screen-design-rules.md)を参照

## Claude Codeへの指示例

ドキュメントを参照させる際の指示例:

```
docs/01-requirements/ の要件定義を確認して、
docs/03-ui-design/ のUI仕様と docs/05-interactions/ のインタラクション仕様に基づいて実装してください。
実装の際は docs/00-rules/ のコーディング規約に従ってください。
```

## 注意事項

- 機密情報を含む場合は、適切なアクセス制御を設定してください
- プロジェクト固有の情報は、このテンプレートをコピー後に編集してください
- ドキュメントの粒度は、プロジェクトの規模に応じて調整してください
